# Paper Folder Structure

## Convention

```
paper/
├── main.tex               # LaTeX entry point — inputs all sections
├── index.md               # Reference index (auto-generated by index-builder agent)
├── docs/                  # Source research PDFs
├── sections_md/           # Markdown drafts — SOURCE OF TRUTH
├── sections/              # Generated .tex files — NEVER EDIT MANUALLY
├── figures/               # Paper figures and diagrams (optional)
└── scripts/
    ├── md_to_tex.sh        # Converts sections_md/*.md → sections/*.tex
    └── export_overleaf.sh  # Regenerates .tex then zips for Overleaf upload
```

---

## Rules

**`sections_md/` is the source of truth.** All writing happens here. The `.tex` files in `sections/` are derived artifacts — regenerated on demand by `md_to_tex.sh`. Never edit `.tex` files manually; your changes will be overwritten.

**`docs/` naming convention.** Name PDF files using their arXiv ID:
```
2309.07430v5.pdf
2502.04413v1.pdf
```
This enables the index-builder agent to fetch metadata automatically from the arXiv API. Non-arXiv papers can use any name — the agent will mark them as `[metadata unavailable — read PDF directly]`.

---

## Adding a New Section

1. Create `sections_md/<name>.md` with a section heading.
2. Add `\input{sections/<name>}` to `main.tex` in the correct position.
3. Run `bash scripts/md_to_tex.sh` to generate `sections/<name>.tex`.

## Bootstrapping a New Paper

Copy this skill's `paper-template/` directory and rename it `paper/` in your project:

```bash
cp -r <skill-folder>/paper-template paper
```

Then run `bash paper/scripts/md_to_tex.sh` after writing each section.

---

## scripts/md_to_tex.sh

Converts all markdown sections to LaTeX via pandoc:

```bash
bash paper/scripts/md_to_tex.sh
```

Requires: `pandoc` installed (`brew install pandoc` or `apt install pandoc`).

## scripts/export_overleaf.sh

Regenerates all `.tex` files, then packages `main.tex + sections/ + figures/` into a timestamped zip:

```bash
cd paper && bash scripts/export_overleaf.sh
```

Output: `overleaf_YYYYMMDD_HHMMSS.zip` in the `paper/` directory.
Upload to Overleaf: **New Project → Upload Project → select the zip**.
